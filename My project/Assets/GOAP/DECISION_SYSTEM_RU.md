# Динамические решения GOAP

Полное описание архитектуры, алгоритма A*, runtime-цикла и применённых паттернов находится в [основном техническом документе](../../../Documentation/ARCHITECTURE_RU.md).

## Goal Score

В инспекторе `GoapGoalDefinition` поле `Base Score` задает постоянную важность цели. В `Fact Score Modifiers` можно добавить один или несколько фактов:

1. Перетащите Fact, например `Hunger`.
2. В `Fact Range` укажите диапазон входного значения, например `65 -> 100`.
3. В `Score Range` укажите вклад в приоритет, например `0 -> 35`.
4. Оставьте `Clamp` включенным, чтобы значение не выходило за заданный диапазон.

Итоговая формула: `Final Score = Base Score + Fact Modifiers + Scorer Components`.

Для нестандартных правил добавьте на NPC компонент, наследующийся от `GoapGoalScorerBehaviour`, и верните дополнительный score из `EvaluateScore`. Runtime Debugger показывает каждый вклад отдельно.

## Hysteresis и cooldown

`Goal Switch Threshold` находится в `GoapAgentProfile`. Новая цель должна превысить текущую минимум на это значение, иначе агент продолжит текущую цель. Это устраняет частое переключение при близких оценках.

`Cooldown (Seconds)` находится в Goal. После достижения цель временно исключается из выбора. В Debugger она помечается как `COOLDOWN` с оставшимся временем.

## Прерывание Action

Поле `Goal Interruption` в Action поддерживает три режима:

- `Immediate`: сразу отменить действие, если выбрана другая цель.
- `Finish Current Action`: закончить текущее действие и затем переключиться.
- `Finish Current Plan`: закончить оставшуюся последовательность текущего плана.

Если предусловия действия или текущая цель стали невалидными, агент все равно отменит небезопасное выполнение и перепланируется.

## Контекстная цель и стоимость

Секция `Planning Context` в Action связывает план с объектом сцены:

- `Automatic`: извлечь категорию из `SmartObject Interaction` или шагов `Find Smart Object` / `Move To Target`.
- `Smart Object Category`: явно найти ближайший объект заданной категории.
- `Named Target`: взять Transform из `GoapAgentAuthoring > Named Targets`.
- `None`: не использовать target при планировании.

`Distance Cost / Unit` добавляет к базовой стоимости расстояние до выбранного объекта. Недоступный обязательный target исключает Action из поиска. Выбранный объект хранится в `GoapActionContext` как `SmartObjectTarget`, `NamedTarget` и общий `Target`, поэтому custom executor может использовать ровно тот объект, стоимость которого оценил planner.

Дополнительные ситуативные штрафы реализуются компонентами от `GoapActionCostProviderBehaviour`. Например, можно учитывать опасность маршрута, перегруженность станции или специализацию NPC.

## Проверка в Runtime Debugger

Откройте `Tools > GOAP > Runtime Debugger` в Play Mode и выберите NPC.

- На вкладке `Goals` цели отсортированы по итоговому score. Внутри каждой цели показаны Base Priority, модификаторы и cooldown.
- На вкладке `Actions` сравните Base Cost и Context Cost. `n/a` означает, что обязательный объект или контекст сейчас недоступен.
- В `History` событие `GoalSwitchDeferred` показывает, когда смена цели была отложена политикой прерывания.
- В визуальном графе выбранная цель и выполняемый план подсвечиваются, а Goal badge содержит фактическую причину выбора.
